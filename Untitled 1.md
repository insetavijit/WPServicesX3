- **1. Definition Phase**
    
    - **What** Define the project goal and success condition
        
    - **Scope**
        
        - **Outcomes** What results will be delivered as part of this project
            
        - **Exclusions** What is explicitly not included
            
    - **Reference**
        
        - **Business objective** The underlying business purpose the project exists to serve
            
        - **Client brief / agreement** The approved document defining requirements, expectations, and constraints
            
        - **Success statement** The sentence that declares when the project is complete


- **2. Breakdown Phase**
    
    - **What** Decompose the project into deliverables, milestones, and tasks
        
    - **Scope**
        
        - **Approved deliverables** Outputs that the client has explicitly agreed to receive
            
        - **Execution boundaries** Work limited strictly to defined tasks and milestones
            
    - **Reference**
        
        - **Deliverables list** The complete inventory of outputs to be produced
            
        - **Milestone plan** The ordered checkpoints grouping related work
            
        - **Task breakdown** The smallest executable actions required for completion


- **3. Execution Phase**
    
    - **What** Perform the planned work according to defined sequence and standards
        
    - **Scope**
        
        - **Execution sequence** Tasks performed only in approved order
            
        - **Quality standards** Predefined rules the work must comply with
            
    - **Reference**
        
        - **Execution SOPs** Step-by-step procedures governing task execution
            
        - **Quality benchmarks** Measurable standards used to judge correctness
            
        - **Change control rules** Conditions under which deviations are allowed
            
- **4. Validation Phase**
    
    - **What** Verify that deliverables meet functional and quality expectations
        
    - **Scope**
        
        - **Validation checks** Functional, performance, and compliance verification
            
        - **Revision limits** Corrections allowed only within approved scope
            
    - **Reference**
        
        - **Acceptance criteria** Binary conditions defining pass or fail
            
        - **QA checklist** The inspection list used during review
            
        - **Review notes** Recorded findings and approved corrections
            
- **5. Acceptance Phase**
    
    - **What** Obtain formal approval and close the project
        
    - **Scope**
        
        - **Final approval only** Confirmation of completion without new work
            
        - **Closure boundary** No additional execution after acceptance
            
    - **Reference**
        
        - **Client sign-off** Written confirmation that requirements are satisfied
            
        - **Handover documents** Access, files, and usage instructions
            
        - **Closure statement** Formal declaration that the project is complete
